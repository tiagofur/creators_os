## How to Use This Documentation

1. **Product Managers**: Start with `01-product/` for feature specs and roadmap
2. **Designers**: Reference `02-design-system/` for all visual decisions
3. **Frontend Developers**: Follow `03-frontend-guidelines/` for implementation
4. **Marketing Team**: Use `04-brand-marketing/` for consistent brand communication

## Principles

- **Single Source of Truth**: All design decisions documented here
- **Living Document**: Updated as the product evolves
- **Platform-Consistent**: Applies to Web, Mobile, and Desktop
- **Creator-First**: Every decision serves content creators
