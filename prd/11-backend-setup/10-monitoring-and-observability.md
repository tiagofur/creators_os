# Monitoring & Observability Specification
## Ordo Creator OS — Go Backend on AWS

**Version:** 1.0  
**Date:** 2026-03-10  
**Status:** Production Design  
**Target:** All environments (dev, staging, production)

---

## Executive Summary

The monitoring & observability strategy for Ordo Creator OS is built on three core pillars: **Logs**, **Metrics**, and **Traces**. The system enables issue diagnosis in under 5 minutes through structured JSON logging, Prometheus-style metrics, and OpenTelemetry distributed tracing. This document provides production-ready implementations in Go with AWS integrations.

---

## 1. Observability Strategy

### Three Pillars

| Pillar | Purpose | Backend | Format |
|--------|---------|---------|--------|
| **Logs** | Event record + debugging context | CloudWatch Logs | Structured JSON |
| **Metrics** | Performance & health quantification | CloudWatch Metrics / Datadog | Prometheus exposition format |
| **Traces** | Request flow across services | AWS X-Ray / Datadog APT | W3C Trace Context |

### Design Goals

1. **Diagnosis in < 5 minutes**: Every critical issue has enough context in logs + metrics to identify root cause
2. **Minimal overhead**: Instrumentation < 5% latency impact
3. **Privacy by default**: No PII, secrets, or sensitive data in observability outputs
4. **Cost-effective**: Intelligent sampling and retention policies
5. **Developer-friendly**: Easy correlation between logs, metrics, and traces

### Observability Contract

Every request generates:
- 1 structured log entry on arrival (DEBUG level)
- 1 structured log entry on completion (INFO/WARN/ERROR based on status)
- 1 request duration metric (histogram bucket)
- 1 trace span tree (with child spans for DB, external APIs, etc.)
- Optional: alert if performance or error thresholds crossed

## 2. Structured Logging

### Library Choice: zerolog

**Why zerolog?**
- Fast JSON logging (10-100x faster than logrus)
- Contextual field attachment
- Leveled output (dev: console, prod: JSON)
- Integrates cleanly with OpenTelemetry
- Zero allocations for disabled log levels

### Log Format

**Development**
```

```

**Production (JSON)**

### Standard Fields on Every Log

```go
type LogContext struct {
    RequestID   string // Unique per request, generated by middleware
    UserID      string // Empty if unauthenticated
    WorkspaceID string // Empty if not workspace-scoped
    Service     string // "ordo-api"
    Method      string // HTTP method or operation type
    Path        string // HTTP path or operation name
    TraceID     string // W3C Trace Context trace-id
}
```

### Log Levels

| Level | When to Use | Example |
|-------|-----------|---------|
| **DEBUG** | Detailed dev-only info | Executed SQL query, cache hit/miss, JWT validation step |
| **INFO** | Notable events in normal operation | Request completed, user signed up, content published |
| **WARN** | Recoverable issues requiring attention | Rate limit approaching, slow query (>200ms), retry succeeded, DB connection timeout |
| **ERROR** | Failures requiring investigation | Database error, AI provider returned error, payment failed, unhandled exception |
| **FATAL** | Startup failures only | Invalid config, cannot connect to database, cannot create tables |

### Logging Rules

**What to Log (Always)**
- Every incoming HTTP request: method, path, remote_addr
- Every outgoing HTTP request: target service, duration, status code
- Request completion: status code, total duration, bytes sent/received
- Authentication events: login attempt, logout, token refresh, password reset
- Business events: user signup, subscription change, content published, remix created
- All errors: error message, error type, stack trace, request context
- Warnings: rate limits approaching, slow database queries (>200ms), external API latency (>1s)

**Never Log (Security/Privacy)**
- Passwords or password hashes
- API keys, OAuth tokens, JWT tokens (log token ID only)
- Credit card numbers or banking details
- Email addresses (except in debug mode)
- Full request/response bodies (log size + first 100 bytes only)
- PII: SSNs, passport numbers, phone numbers (unless critical for incident response)

### Go Implementation: zerolog Setup

```go
// pkg/logger/logger.go
package logger

import (
	"context"
	"io"
	"os"
	"time"

	"github.com/rs/zerolog"
	"github.com/rs/zerolog/log"
)

type ContextKey string

const (
	RequestIDKey   ContextKey = "request_id"
	UserIDKey      ContextKey = "user_id"
	WorkspaceIDKey ContextKey = "workspace_id"
	TraceIDKey     ContextKey = "trace_id"
)

// Init sets up the global logger based on environment.
func Init(env string) {
	output := io.Writer(os.Stdout)

	// Development: pretty console output
	if env == "development" {
		output = zerolog.ConsoleWriter{
			Out:        os.Stdout,
			TimeFormat: "15:04:05",
			NoColor:    false,
		}
		zerolog.SetGlobalLevel(zerolog.DebugLevel)
	} else {
		// Production: JSON output
		zerolog.SetGlobalLevel(zerolog.InfoLevel)
	}

	log.Logger = zerolog.New(output).
		With().
		Timestamp().
		Str("service", "ordo-api").
		Logger()
}

// WithContext attaches request context to logger.
func WithContext(ctx context.Context) *zerolog.Logger {
	logger := log.With()

	if requestID, ok := ctx.Value(RequestIDKey).(string); ok && requestID != "" {
		logger = logger.Str("request_id", requestID)
	}
	if userID, ok := ctx.Value(UserIDKey).(string); ok && userID != "" {
		logger = logger.Str("user_id", userID)
	}
	if workspaceID, ok := ctx.Value(WorkspaceIDKey).(string); ok && workspaceID != "" {
		logger = logger.Str("workspace_id", workspaceID)
	}
	if traceID, ok := ctx.Value(TraceIDKey).(string); ok && traceID != "" {
		logger = logger.Str("trace_id", traceID)
	}

	l := logger.Logger()
	return &l
}

// ContextWithValues attaches logging values to context.
func ContextWithValues(ctx context.Context, requestID string, userID string, workspaceID string, traceID string) context.Context {
	if requestID != "" {
		ctx = context.WithValue(ctx, RequestIDKey, requestID)
	}
	if userID != "" {
		ctx = context.WithValue(ctx, UserIDKey, userID)
	}
	if workspaceID != "" {
		ctx = context.WithValue(ctx, WorkspaceIDKey, workspaceID)
	}
	if traceID != "" {
		ctx = context.WithValue(ctx, TraceIDKey, traceID)
	}
	return ctx
}
```

## 3. Request Logging Middleware

### Middleware Behavior

- **On Request Start** (DEBUG): method, path, remote_addr
- **On Request End** (INFO/WARN/ERROR): method, path, status, duration, request_id, user_id
- **Slow Request** (WARN): if duration > 500ms
- **Server Error** (ERROR): if status >= 500

### Go Implementation

```go
// pkg/middleware/request_logging.go
package middleware

import (
	"context"
	"fmt"
	"net"
	"net/http"
	"strings"
	"time"

	"github.com/google/uuid"
	"github.com/rs/zerolog"
	"ordo/pkg/logger"
)

// RequestLoggingMiddleware logs incoming and outgoing HTTP requests.
func RequestLoggingMiddleware(next http.Handler) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		// Generate unique request ID
		requestID := r.Header.Get("X-Request-ID")
		if requestID == "" {
			requestID = uuid.New().String()[:8] // Short format: abc12345
		}

		// Extract or generate trace ID from W3C Trace Context header
		traceID := extractTraceID(r)
		if traceID == "" {
			traceID = uuid.New().String()
		}

		// Extract user ID from context (set by auth middleware)
		userID := r.Header.Get("X-User-ID")
		workspaceID := r.Header.Get("X-Workspace-ID")

		// Attach to context
		ctx := logger.ContextWithValues(r.Context(), requestID, userID, workspaceID, traceID)
		r = r.WithContext(ctx)

		// Log request start (DEBUG level)
		remoteAddr := getRemoteAddr(r)
		logger.WithContext(ctx).Debug().
			Str("method", r.Method).
			Str("path", r.URL.Path).
			Str("remote_addr", remoteAddr).
			Msg("request start")

		// Capture response writer
		wrapped := &responseWriter{
			ResponseWriter: w,
			statusCode:     http.StatusOK,
			bodySize:       0,
		}

		// Record start time
		startTime := time.Now()

		// Call next handler
		next.ServeHTTP(wrapped, r)

		// Log request end with timing and status
		duration := time.Since(startTime)
		durationMs := duration.Milliseconds()

		logLevel := zerolog.InfoLevel
		if wrapped.statusCode >= 500 {
			logLevel = zerolog.ErrorLevel
		} else if wrapped.statusCode >= 400 {
			logLevel = zerolog.WarnLevel
		}

		// Warn on slow requests
		if durationMs > 500 {
			logLevel = zerolog.WarnLevel
		}

		logEvent := logger.WithContext(ctx).WithLevel(logLevel)
		if wrapped.statusCode >= 500 {
			logEvent = logger.WithContext(ctx).Error()
		} else if logLevel == zerolog.WarnLevel && durationMs > 500 {
			logEvent = logger.WithContext(ctx).Warn()
		} else {
			logEvent = logger.WithContext(ctx).Info()
		}

		logEvent.
			Str("method", r.Method).
			Str("path", r.URL.Path).
			Int("status", wrapped.statusCode).
			Int64("duration_ms", durationMs).
			Int("bytes_out", wrapped.bodySize).
			Msg("request completed")
	})
}

// responseWriter wraps http.ResponseWriter to capture status and body size.
type responseWriter struct {
	http.ResponseWriter
	statusCode int
	bodySize   int
}

func (rw *responseWriter) WriteHeader(code int) {
	rw.statusCode = code
	rw.ResponseWriter.WriteHeader(code)
}

func (rw *responseWriter) Write(b []byte) (int, error) {
	n, err := rw.ResponseWriter.Write(b)
	rw.bodySize += n
	return n, err
}

// extractTraceID extracts trace ID from W3C Trace Context header.
// Format: traceparent: 00-0af7651916cd43dd8448eb211c80319c-b7ad6b7169203331-01
func extractTraceID(r *http.Request) string {
	traceparent := r.Header.Get("traceparent")
	if traceparent == "" {
		return ""
	}
	parts := strings.Split(traceparent, "-")
	if len(parts) >= 2 {
		return parts[1] // trace-id is the second part
	}
	return ""
}

// getRemoteAddr extracts client IP from X-Forwarded-For or RemoteAddr.
func getRemoteAddr(r *http.Request) string {
	if xff := r.Header.Get("X-Forwarded-For"); xff != "" {
		// X-Forwarded-For can contain multiple IPs; take the first
		if parts := strings.Split(xff, ","); len(parts) > 0 {
			return strings.TrimSpace(parts[0])
		}
	}
	if xri := r.Header.Get("X-Real-IP"); xri != "" {
		return xri
	}
	host, _, _ := net.SplitHostPort(r.RemoteAddr)
	return host
}
```

```go
// In middleware/request_logging.go - record metrics
func RequestLoggingMiddleware(metricsReg *metrics.Metrics) func(http.Handler) http.Handler {
	return func(next http.Handler) http.Handler {
		return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
			// Increment in-flight requests
			metricsReg.HTTPRequestsInFlight.Inc()
			defer metricsReg.HTTPRequestsInFlight.Dec()

			// ... existing code ...

			startTime := time.Now()
			wrapped := &responseWriter{ResponseWriter: w, statusCode: http.StatusOK}

			next.ServeHTTP(wrapped, r)

			duration := time.Since(startTime).Seconds()

			// Record metrics
			metricsReg.HTTPRequestsTotal.WithLabelValues(
				r.Method,
				r.URL.Path,
				fmt.Sprintf("%d", wrapped.statusCode),
			).Inc()

			metricsReg.HTTPRequestDuration.WithLabelValues(
				r.Method,
				r.URL.Path,
			).Observe(duration)

			metricsReg.HTTPRequestSize.WithLabelValues(
				r.Method,
				r.URL.Path,
			).Observe(float64(r.ContentLength))

			metricsReg.HTTPResponseSize.WithLabelValues(
				r.Method,
				r.URL.Path,
			).Observe(float64(wrapped.bodySize))

			// ... existing logging code ...
		})
	}
}
```

## 5. Health Check Endpoints

### GET /health (Liveness Probe)

Returns 200 if the process is running. No dependency checks. Used by ECS task health check and Kubernetes liveness probes.

### GET /ready (Readiness Probe)

Checks all critical dependencies (PostgreSQL, Redis). Returns 200 if healthy, 503 if unhealthy. Used by ALB target group health check.

### GET /metrics (Prometheus Exposition)

Prometheus-format metrics. Protected to internal network only.

### Go Implementation

```go
// pkg/health/health.go
package health

import (
	"context"
	"database/sql"
	"encoding/json"
	"net/http"
	"time"

	"github.com/redis/go-redis/v9"
	"github.com/rs/zerolog/log"
)

type HealthChecker struct {
	db    *sql.DB
	redis *redis.Client
}

func New(db *sql.DB, redis *redis.Client) *HealthChecker {
	return &HealthChecker{
		db:    db,
		redis: redis,
	}
}

// LivenessResponse is returned by GET /health
type LivenessResponse struct {
	Status string `json:"status"` // "ok"
}

// ReadinessResponse is returned by GET /ready
type ReadinessResponse struct {
	Status   string                     `json:"status"` // "ok" or "unhealthy"
	Database ReadinessComponentResponse `json:"database"`
	Redis    ReadinessComponentResponse `json:"redis"`
}

type ReadinessComponentResponse struct {
	Status  string `json:"status"`   // "ok" or "error"
	Details string `json:"details,omitempty"`
}

// HandleLiveness returns 200 if process is running.
func (hc *HealthChecker) HandleLiveness(w http.ResponseWriter, r *http.Request) {
	w.Header().Set("Content-Type", "application/json")
	w.WriteHeader(http.StatusOK)
	json.NewEncoder(w).Encode(LivenessResponse{Status: "ok"})
}

// HandleReadiness checks dependencies and returns 200/503.
func (hc *HealthChecker) HandleReadiness(w http.ResponseWriter, r *http.Request) {
	ctx, cancel := context.WithTimeout(r.Context(), 5*time.Second)
	defer cancel()

	resp := ReadinessResponse{
		Status: "ok",
	}

	// Check PostgreSQL
	if err := hc.db.PingContext(ctx); err != nil {
		resp.Status = "unhealthy"
		resp.Database.Status = "error"
		resp.Database.Details = err.Error()
		log.Warn().Err(err).Msg("database readiness check failed")
	} else {
		resp.Database.Status = "ok"
	}

	// Check Redis
	if err := hc.redis.Ping(ctx).Err(); err != nil {
		resp.Status = "unhealthy"
		resp.Redis.Status = "error"
		resp.Redis.Details = err.Error()
		log.Warn().Err(err).Msg("redis readiness check failed")
	} else {
		resp.Redis.Status = "ok"
	}

	// If any check failed, return 503
	statusCode := http.StatusOK
	if resp.Status == "unhealthy" {
		statusCode = http.StatusServiceUnavailable
	}

	w.Header().Set("Content-Type", "application/json")
	w.WriteHeader(statusCode)
	json.NewEncoder(w).Encode(resp)
}
```

## 6. Distributed Tracing

### Library: OpenTelemetry Go SDK

**Why OpenTelemetry?**
- Vendor-neutral standard (works with X-Ray, Datadog, Jaeger, etc.)
- Automatic instrumentation for HTTP, database, gRPC
- W3C Trace Context propagation
- Minimal performance overhead

### Trace Context Propagation

We use **W3C Trace Context** header format:
```
traceparent: 00-0af7651916cd43dd8448eb211c80319c-b7ad6b7169203331-01
            |  |                               |                 |  |
            |  |                               |                 |  +-- trace-flags (01=sampled)
            |  |                               +-- parent-span-id
            |  +-- trace-id
            +-- version (00=current)
```

### Automatic Instrumentation

**HTTP Requests**: Every request automatically gets a span
```
Span: POST /api/contents [duration: 145ms]
  ├─ Span: SELECT users WHERE id=$1 [duration: 5ms]
  ├─ Span: POST https://api.openai.com/v1/chat/completions [duration: 1200ms]
  └─ Span: PUT s3://ordo-content-bucket/idea-abc123.json [duration: 30ms]
```

**Database Queries**: pgx driver auto-instruments
```
Span: Query: INSERT INTO ideas (...)
  └─ [duration: 12ms, rows: 1]
```

### Go Implementation: OpenTelemetry Setup

```go
// pkg/tracing/tracing.go
package tracing

import (
	"context"
	"fmt"

	"go.opentelemetry.io/otel"
	"go.opentelemetry.io/otel/exporters/otlp/otlptrace/otlptracehttp"
	"go.opentelemetry.io/otel/sdk/resource"
	sdktrace "go.opentelemetry.io/otel/sdk/trace"
	semconv "go.opentelemetry.io/otel/semconv/v1.21.0"
	"go.opentelemetry.io/otel/trace"
)

// Init initializes OpenTelemetry tracing.
// For production, export to X-Ray or Datadog APT.
func Init(serviceName, env, version string) (func(context.Context) error, error) {
	// Create exporter (configure based on env)
	var exporter sdktrace.SpanExporter
	var err error

	if env == "production" || env == "staging" {
		// Export to AWS X-Ray via OTLP HTTP
		exporter, err = otlptracehttp.New(
			context.Background(),
			otlptracehttp.WithEndpoint("localhost:4318"), // X-Ray daemon on localhost
		)
		if err != nil {
			return nil, fmt.Errorf("create otlp exporter: %w", err)
		}
	} else {
		// Development: use stdout exporter
		exporter, err = otlptracehttp.New(
			context.Background(),
			otlptracehttp.WithEndpoint("localhost:4317"), // Or use file exporter
		)
		if err != nil {
			return nil, fmt.Errorf("create dev exporter: %w", err)
		}
	}

	// Create resource
	res, err := resource.New(context.Background(),
		resource.WithAttributes(
			semconv.ServiceNameKey.String(serviceName),
			semconv.ServiceVersionKey.String(version),
			semconv.DeploymentEnvironmentKey.String(env),
		),
	)
	if err != nil {
		return nil, fmt.Errorf("create resource: %w", err)
	}

	// Create tracer provider
	tp := sdktrace.NewTracerProvider(
		sdktrace.WithBatcher(exporter),
		sdktrace.WithResource(res),
		// Sample 100% of traces in production (or adjust based on volume)
		sdktrace.WithSampler(sdktrace.AlwaysSample()),
	)

	// Set global tracer provider
	otel.SetTracerProvider(tp)

	// Return shutdown function
	return tp.Shutdown, nil
}

// Tracer returns a tracer for the given package.
func Tracer(packageName string) trace.Tracer {
	return otel.Tracer(packageName)
}
```

### Manual Span Creation

```go
// pkg/service/content.go
package service

import (
	"context"

	"go.opentelemetry.io/otel/attribute"
	"go.opentelemetry.io/otel/codes"
	"ordo/pkg/tracing"
)

type ContentService struct {
	// ... fields ...
}

func (cs *ContentService) PublishContent(ctx context.Context, contentID string) error {
	// Create span for this operation
	tracer := tracing.Tracer("service.content")
	ctx, span := tracer.Start(ctx, "PublishContent")
	defer span.End()

	// Add attributes to span
	span.SetAttributes(
		attribute.String("content_id", contentID),
	)

	// Call AI provider (automatically traced via instrumentation)
	ideas, err := cs.aiProvider.GenerateIdeas(ctx, contentID)
	if err != nil {
		span.RecordError(err)
		span.SetStatus(codes.Error, "failed to generate ideas")
		return err
	}

	span.AddEvent("ideas_generated", trace.WithAttributes(
		attribute.Int("count", len(ideas)),
	))

	// Publish to S3 (automatically traced)
	if err := cs.s3.Upload(ctx, "ideas-"+contentID, ideas); err != nil {
		span.RecordError(err)
		span.SetStatus(codes.Error, "failed to upload to s3")
		return err
	}

	span.SetStatus(codes.Ok, "")
	return nil
}
```

### HTTP Middleware with Trace Context

```go
// pkg/middleware/tracing.go
package middleware

import (
	"net/http"

	"go.opentelemetry.io/otel"
	"go.opentelemetry.io/otel/propagation"
	"go.opentelemetry.io/otel/trace"
)

// TracingMiddleware adds distributed tracing to HTTP requests.
func TracingMiddleware(serviceName string) func(http.Handler) http.Handler {
	return func(next http.Handler) http.Handler {
		return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
			// Extract trace context from request
			propagator := propagation.NewCompositeTextMapPropagator(
				propagation.TraceContext{},
				propagation.Baggage{},
			)
			ctx := propagator.Extract(r.Context(), propagation.HeaderCarrier(r.Header))

			// Create span for this request
			tracer := otel.Tracer(serviceName)
			ctx, span := tracer.Start(
				ctx,
				r.Method+" "+r.URL.Path,
				trace.WithSpanKind(trace.SpanKindServer),
			)
			defer span.End()

			// Set span attributes
			span.SetAttributes(
				attribute.String("http.method", r.Method),
				attribute.String("http.url", r.URL.String()),
				attribute.String("http.target", r.URL.Path),
				attribute.String("http.host", r.Host),
				attribute.String("http.scheme", r.URL.Scheme),
			)

			// Call next handler with new context
			r = r.WithContext(ctx)
			next.ServeHTTP(w, r)
		})
	}
}
```

| **Zero Healthy Tasks** | Healthy ECS tasks = 0 | 1 minute | Page on-call |

5. Identify affected endpoint (path with most errors)

**Root Cause Scenarios**
- **Database is slow/down**: Check AWS RDS dashboard → Instance metrics
- **External API down** (OpenAI, Stripe): Check provider status pages
- **Memory leak**: Check ECS task memory usage graph
- **Configuration error**: Check recent deployments in CodePipeline
- **High traffic spike**: Check request rate metric
- **Service dependency down**: Check Redis, PostgreSQL readiness in /ready

**Remediation**
- If database: Restart RDS instance or check slow query logs
- If external API: Degrade gracefully (return cached data or error)
- If memory leak: Restart ECS tasks (will be auto-replaced)
- If config: Rollback to previous deployment
- If traffic spike: Scale up ECS task count via ASG

**Escalation**
- If not resolved in 10 minutes: Page on-call engineer
- Prepare: error logs, dashboard screenshot, recent changes

### Runbook 2: Slow Database Queries (>200ms)

**Triggered By:** Manual investigation or warning alert

**Diagnosis**
1. Open CloudWatch Insights:
   ```sql
   fields @timestamp, duration_ms, operation, user_id
   | filter duration_ms > 200
   | stats avg(duration_ms), max(duration_ms), count() by operation
   | sort max(duration_ms) desc
   ```
2. Identify slowest query operations
3. Log into AWS RDS console → Performance Insights
4. Check query execution plans for N+1 queries or full table scans

**Common Causes**
- **Missing index**: Query doing full table scan
- **Too many JOIN operations**: Excessive operations on related tables
- **N+1 problem**: Fetching parent, then looping through children querying each
- **Connection pool exhaustion**: Queries waiting for available connection
- **Row count explosion**: Table grew to millions of rows
- **Statistics stale**: Query planner using incorrect selectivity estimates

**Remediation**
- Add index: `CREATE INDEX idx_users_workspace_id ON users(workspace_id)`
- Refactor query to use JOIN instead of loop
- Update table statistics: `ANALYZE table_name`
- Increase connection pool size: `max_connections` in PostgreSQL
- Archive old data to separate table
- Scale up RDS instance (more CPU/memory)

**Prevention**
- Add slow query logging to staging database
- Monitor query performance trends weekly
- Code review: check for N+1 queries in new code

### Runbook 3: AI Provider Outage (OpenAI/Anthropic)

**Triggered By:** Manual detection or alert (error rate >10% for provider)

**Diagnosis**
1. Check provider status page:
   - OpenAI: https://status.openai.com
   - Anthropic: Check #outages Slack channel
2. Run CloudWatch query:
   ```sql
   fields @timestamp, provider, status, duration_ms
   | filter message like /external_api/ and provider = "openai"
   | stats count() as errors by status
   ```
3. Confirm error rate is above threshold

**User Impact**
- **Generate ideas**: Returns error or cached ideas
- **Remix content**: Queue jobs fail; user sees "Try again later"
- **Smart suggestions**: Disabled; show generic suggestions instead

**Remediation (Graceful Degradation)**
1. Enable feature flags to disable AI features
   ```go
   if featureFlag.IsEnabled("disable_ai_generation") {
       return cachedIdeas, nil
   }
   ```
2. Return cached results from previous successful calls
3. Queue failed jobs for retry when provider recovers
4. Show user-friendly message: "AI features temporarily unavailable"
5. Monitor provider status page for recovery
6. Disable degradation once provider returns to normal

**Communication**
- Post to #status Slack channel: "AI features temporarily unavailable due to provider outage"
- Check internal dashboard every 5 minutes for recovery
- Update status page when provider recovers

### Runbook 4: Memory Leak in Go Application

**Triggered By:** ECS task restarting frequently or memory usage at 90%+

**Diagnosis**
1. Check ECS dashboard → Task → Memory usage graph (should be flat, not climbing)
2. SSH into container (if allowed):
   ```bash
   curl http://localhost:8080/debug/pprof/heap > heap.prof
   go tool pprof heap.prof
   (pprof) top10  # Show top 10 memory allocators
   ```
3. Look for growing allocations (should be stable over time)
4. Generate heap graph:
   ```bash
   (pprof) svg  # Opens SVG in browser showing allocation tree
   ```

**Common Causes**
- **Unbounded slice growth**: Slice append in loop without cap check
- **Unreleased goroutines**: Goroutine pool goroutines never exit
- **Context not cancelled**: Long-lived context holding references
- **Circular references**: Objects reference each other preventing GC
- **Global cache unbounded**: Cache size grows indefinitely
- **Channel not drained**: Goroutine blocking on channel send

**Remediation**
1. Review recent code changes to memory-intensive services
2. Profile on staging with production-like load
3. Identify allocation hotspot using pprof
4. Fix: add bounds checking, cancel contexts, drain channels
5. Restart affected ECS tasks (temporary fix)
6. Deploy fix; monitor memory trend

### Runbook 5: WebSocket Disconnections (Mass Reconnections)

**Triggered By:** Alert on active_websocket_connections dropping >50% in 1 minute

**Diagnosis**
1. Check CloudWatch: WebSocket connection count graph
2. Correlate with deployment time (deployment may have caused restarts)
3. Check application logs:
   ```sql
   fields @timestamp, message, user_id, error_type
   | filter message like /websocket|disconnect/
   | stats count() by error_type
   ```
4. Identify error pattern (EOF, timeout, protocol error, etc.)

**Common Causes**
- **ALB timeout**: ALB killed connection after 60s idle (default)
- **Goroutine leak**: Server running out of goroutines (ulimit)
- **TCP backlog full**: Connection queue full; OS dropping connections
- **Memory pressure**: Go GC pausing all threads (causing timeout)
- **Load balancer config change**: Reduced idle timeout

**Remediation**
- **ALB timeout**: Increase idle timeout to 300s in target group settings
- **Goroutine leak**: Restart ECS tasks; profile for leaks
- **TCP backlog**: Increase listen(2) backlog: `net.ListenAndServe(":8080", ...)`
- **Memory pressure**: Scale up ECS task memory or reduce connections per task
- **Config change**: Revert ALB idle timeout change

**Prevention**
- Send WebSocket PING every 30 seconds to keep connection alive
- Implement exponential backoff for client reconnection
- Monitor WebSocket connection trends; alert on anomalies

### Runbook 6: Queue Backup (>1000 Jobs Pending)

**Triggered By:** Alert when `queue_depth > 1000`

**Diagnosis**
1. Check queue depth metric graph (is it climbing or stable?)
2. Check job processing rate:
   ```sql
   fields @timestamp, type, status
   | filter message like /queue_job/
   | stats count() as completed by bin(1m), type
   ```
3. Compare: are we processing slower than jobs arriving?
4. Check if specific job type is stuck:
   ```sql
   fields @timestamp, type, duration_ms, status
   | filter message like /queue_job/ and type = "remix"
   | stats avg(duration_ms), max(duration_ms) by status
   ```

**Root Causes**
- **Job handler deadlocked**: Worker goroutine stuck on I/O
- **External API slow**: All jobs waiting on slow API (e.g., OpenAI)
- **Database connection limit**: Jobs can't acquire DB connection
- **Insufficient workers**: Not enough goroutines processing jobs
- **Job handler timeout**: Handler hanging, not returning

**Remediation**
- **Deadlocked handler**: Restart worker pods/ECS tasks
- **Slow external API**: Enable graceful degradation; skip external API calls
- **Database connections**: Increase connection pool size or reduce jobs per worker
- **Insufficient workers**: Scale up worker replicas/ECS tasks
- **Handler timeout**: Add context timeout to handlers; investigate slow operation

**Monitoring**
- Track job processing rate vs arrival rate
- Alert if processing rate drops below 10 jobs/sec
- Alert if max job age > 1 hour
